# 漫游手账 · 迭代路线图（Roadmap）

> 配套文档：`PRD.md`
> 版本：v1.0
> 规划周期：约 5 个版本（v1.0 已交付，v1.1 ~ v3.0 为规划）

---

## 一、路线图总览

```
v1.0 ✅ 已交付      v1.1 🎯 体验补全      v1.2 🚀 核心兑现       v2.0 🌐 账户化         v3.0 ✨ 增值
（MVP 骨架）        （地图+图片）         （真AI+灵感闭环）      （登录+同步+分享）     （差旅/足迹/出行中）
─────────────     ─────────────       ─────────────        ─────────────        ─────────────
基础三件套          地图视图              真实大模型接入          用户系统               差旅报销模式
AI/灵感 Mock       照片上传              POI 联想/兜底          PostgreSQL            旅行足迹地图
活泼 UI            拖拽排序              灵感→行程闭环           多设备同步             年度回顾
本地 SQLite        冲突检测              AI 局部重生成           分享链接/导出长图       出行中"今日"视图
```

**主线逻辑**：先补**体验硬伤**（v1.1）→ 再兑现**核心卖点**（v1.2）→ 然后**账户化**释放数据价值（v2.0）→ 最后做**差异化与情感化增值**（v3.0）。

---

## 二、版本详述

### ✅ v1.0 — MVP（已交付）

**目标**：跑通"规划-预算-记录"基础闭环，验证产品形态。

| 已交付 |
|--------|
| 旅行管理 CRUD（旅行/差旅/周末三类型） |
| 每日规划（行程项增删改、打卡） |
| 预算花销（记账、预算进度、分类汇总、超支提醒） |
| 旅行记录（图文手账、心情） |
| AI 行程推荐骨架（Mock 兜底 + Key 可配置 + 一键存为旅行） |
| 灵感发现骨架（Mock 卡片 + 搜索） |
| 仪表盘统计、活泼多彩响应式 UI |
| Express + sql.js(SQLite) 全栈、本地持久化、可推 GitHub |

---

### 🎯 v1.1 — 体验补全（约 2-3 周）

**目标**：补齐旅行类产品的体验硬伤，让"规划"和"记录"真正好用。
**主题**：地图 + 图片 + 编辑体验

> 执行顺序调整说明：F-NOTE-02（照片上传）性价比高、无外部依赖，已提前完成，详见状态列。

| 优先级 | 需求编号 | 内容 | 说明 | 状态 |
|--------|---------|------|------|------|
| P0 | F-PLAN-03 | 每日规划**地图视图** | 接入高德/Mapbox，点位标注 + 当天连线 + 直线距离 | ⬜ |
| P0 | F-PLAN-06 | 地点 **POI 搜索联想** | 输入地名自动补全坐标与地址 | ⬜ |
| P0 | F-NOTE-02 | **照片上传**与展示 | 多图、压缩、缩略图、灯箱预览（本地压缩为 base64，v2 迁对象存储） | ✅ |
| P1 | F-PLAN-04 | 行程项**拖拽排序 / 跨天移动** | 提升编辑效率 | ✅ |
| P1 | F-PLAN-05 | **时间冲突检测** | 同时段重叠提醒 | ✅ |
| P1 | F-TRIP-02 | 旅行状态**自动流转** | 按当前日期判断进行中/已完成 | ⬜ |

**里程碑验收**
- 任选一天能在地图上看到所有点位与连线。
- 记录可上传并展示至少 3 张图片。
- 行程项可拖拽重排，时间冲突有提示。

**技术要点**：数据模型为 `Activity` 增加 `lat/lng/address/poiId`；引入地图 SDK 与 POI 搜索 API；前端引入拖拽库（如 dnd-kit）。

---

### 🚀 v1.2 — 核心卖点兑现（约 3-4 周）

**目标**：把"AI 推荐"和"灵感发现"从骨架做成真正有价值的功能。
**主题**：真 AI + 内容闭环

> 执行顺序调整说明：AI 相关几项（F-AI-04/05/06）成本低、风险独立、价值可量化，已提前于地图等 v1.1 任务完成，详见下方状态列。

| 优先级 | 需求编号 | 内容 | 说明 | 状态 |
|--------|---------|------|------|------|
| P0 | F-AI-02 | **接入真实大模型** | OpenAI 兼容（可选 DeepSeek/通义），输出与目的地强相关 | 🟡 调用链路已就绪，待配置正式 Key 验证 |
| P1 | F-AI-05 | **POI 规则推荐兜底** | 无 Key 时基于真实 POI 库生成，不再"放之四海皆准" | ✅ 已完成（内置 24 个热门城市 POI 库） |
| P1 | F-AI-04 | AI **局部重新生成** | "换掉第 2 天""更轻松""多点美食" | ✅ 已完成 |
| P1 | F-DISC-02 | 灵感 **→ 一键导入行程** | 内容到行动的转化闭环 | ⬜ |
| P1 | F-DISC-04 | 灵感接**合规内容数据源** | 自有内容库 / 官方合作（替换 Mock） | ⬜ |
| P2 | F-AI-06 | AI 行程**带预估花费**并同步预算 | 规划与预算打通 | ✅ 已完成 |

**里程碑验收**
- 同一目的地、不同偏好，AI 输出有明显差异（采纳率指标可量化）。
- 灵感卡片可一键变成一份可编辑的行程草案。

**技术要点**：Prompt 工程 + 结果缓存（按"目的地+偏好"缓存降成本）；速率/额度限制；POI 库建设。

> ⚠️ 合规红线：灵感内容必须走合法授权渠道，严禁爬取第三方平台。

### 🌐 v2.0 — 账户化与数据价值（约 4-5 周）

**目标**：兑现"后端 + 数据库"的真正价值——多设备、可分享。
**主题**：用户系统 + 同步 + 分享

| 优先级 | 需求编号 | 内容 | 说明 |
|--------|---------|------|------|
| P1 | F-ACC-01 | 用户**注册/登录** | 邮箱 + 第三方（微信/Apple 可选） |
| P1 | F-ACC-02 | 数据按用户隔离、**多设备同步** | 各实体加 `userId` |
| P1 | F-ACC-03 | 数据库升级 **PostgreSQL** | 支持并发与多用户 |
| P1 | F-SHARE-01 | 行程**只读分享链接** | 不可枚举 token + 过期控制 |
| P1 | F-SHARE-02 | 行程/手账**导出长图 / PDF** | 旅行类高传播形态 |
| P2 | F-SHARE-03 | 分享落地页"**复制此行程**" | 拉新转化 |
| - | - | 图片迁移至**对象存储**（OSS/S3）+ CDN | 配合 v1.1 的本地图片 |

**里程碑验收**
- 同一账号在两台设备数据一致。
- 生成的分享链接他人可只读查看，并能一键复制为自己的行程。

**技术要点**：JWT/Session 鉴权；数据迁移脚本（SQLite→PG）；分享 token 安全设计；导出用服务端渲染或前端 canvas 截图。

## 三、优先级矩阵（影响 × 成本）

```
高影响
  │  [真AI接入]          [地图视图]
  │  [照片上传]          [用户系统+同步]
  │  [灵感→行程闭环]
  │  ────────────────────────────────
  │  [拖拽排序]          [差旅报销]
  │  [冲突检测]          [足迹地图]
  │  [分享链接]          [年度回顾]
低影响                                    高成本
  低成本 ───────────────────────────►
```

**先做"高影响低成本"**（真 AI、照片上传、灵感闭环），**再啃"高影响高成本"**（地图、账户同步）。
