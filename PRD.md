## 2. 产品概述

### 2.1 一句话定位
帮助个人用户**轻松规划、智能推荐并温暖记录**每一段旅程（旅行 / 差旅 / 周末出行）的全栈工具。

### 2.2 产品价值主张
- **规划省心**：行程、预算、住宿、交通集中管理，AI 一键生成路线。
- **过程顺手**：出行中随时查看"今天该干嘛"，照片随手记。
- **回忆留存**：图文手账 + 旅行足迹，把旅程变成可回看的资产。

### 2.3 产品边界（本期不做）
- 不做机票 / 酒店 / 门票的**预订与交易**（仅做规划与记录，必要时跳转外部）。
- 不做**社交社区**（关注、私信、UGC 广场），灵感发现仅作内容消费入口。
- 不做**多人协同实时编辑**（v3 阶段评估）。

## 3. 用户与场景

### 3.1 目标用户画像

| 画像 | 特征 | 核心诉求 |
|------|------|---------|
| **休闲旅行者**（主） | 一年 2-4 次中长途旅行，爱拍照爱记录 | 行程规划清晰、回忆可沉淀 |
| **商务差旅者** | 频繁出差，关注效率与报销 | 行程紧凑、费用归集与报销 |
| **周末玩家** | 高频近郊短途，决策快 | 快速找地方、轻量规划 |

### 3.2 核心用户故事（User Stories）

> 格式：作为 **〈角色〉**，我希望 **〈功能〉**，以便 **〈价值〉**。

- **US-01** 作为休闲旅行者，我希望输入目的地和天数就能得到一份合理的行程，以便省去查攻略的时间。
- **US-02** 作为旅行者，我希望在地图上看到每天的行程点位和路线，以便判断安排是否顺路。
- **US-03** 作为旅行者，我希望随手把照片贴进当天的记录里，以便图文并茂地留住回忆。
- **US-04** 作为旅行者，我希望旅行结束后看到"今年去了几个城市"的足迹地图，以便获得成就感。
- **US-05** 作为差旅者，我希望按差旅模板快速登记费用并标记可报销项，以便回公司一键导出报销单。
- **US-06** 作为周末玩家，我希望基于当前位置推荐 2 小时车程内的好去处，以便快速决定周末去哪。
- **US-07** 作为旅行者，我希望把规划好的行程生成一张分享图或链接发给同行的朋友，以便协同确认。
- **US-08** 作为用户，我希望我的数据能在手机和电脑间同步，以便随时随地查看与编辑。

### 3.3 关键场景流（出行三阶段）

```
出行前（规划）           出行中（执行）            出行后（沉淀）
─────────────         ─────────────          ─────────────
创建旅行                 今日行程卡片             写图文手账
AI 生成 / 灵感导入       打卡完成                 整理花销 / 报销
排期 + 地图校验          随手记账 / 拍照           生成回忆 & 足迹
预算设定                 实时查看预算余额          分享旅程
```

## 4. 产品目标与成功指标

### 4.1 北极星指标
**"完成一次完整旅程闭环"的用户数**（创建旅行 → 规划 → 记录 ≥1 条 → 旅程结束）。

### 4.2 分层指标（北极星拆解）

| 阶段 | 指标 | 说明 |
|------|------|------|
| 获取 | 新增旅行创建数 | 用户是否愿意开始 |
| 激活 | 创建后 24h 内添加 ≥3 个行程项的比例 | 是否真正用起来 |
| 留存 | 旅行进行中"今日行程"日活 | 出行中是否依赖产品 |
| 留存 | 次月再创建旅行比例 | 长期价值 |
| 转化 | 灵感卡片 → 导入行程转化率 | 内容到行动的闭环效率 |
| 推荐 | 行程分享次数 / 分享带来的新用户 | 自传播能力 |

### 4.3 北极星之外的体验指标
- AI 推荐"采纳率"（生成后保存为旅行的比例）≥ 30%
- 记录功能含图片的比例 ≥ 50%
- 行程项地图标注率 ≥ 60%

## 5. 竞品分析

| 产品 | 强项 | 我们的差异化机会 |
|------|------|----------------|
| 马蜂窝 / 携程攻略 | 内容海量、预订闭环 | 它们重"逛"，我们重"自己的行程管理与记录" |
| 飞猪行程助手 | 与订单打通 | 我们不绑交易，纯个人化、轻量、隐私友好 |
| Google Trips（已下线）/ TripIt | 行程聚合、邮件解析 | 国内化 + AI 生成 + 手账记录情感化 |
| 小红书 | 灵感与种草 | 我们做"种草后能直接落地成行程"的承接 |
| Notion 旅行模板 | 灵活 | 我们更专注、开箱即用、有地图与 AI |

**结论**：定位差异点 = **「个人化的行程管理 + AI 规划 + 情感化记录」三合一，且不碰交易、隐私友好**。

## 6. 功能需求详述

> 优先级定义：**P0** = 核心必备（不做则产品不成立）；**P1** = 重要（影响留存/转化）；**P2** = 增强（差异化/情感化）。
> 状态定义：✅ 已实现 / 🟡 部分实现 / ⬜ 未实现。

### 6.1 旅行管理（P0）

| 编号 | 需求 | 优先级 | 状态 |
|------|------|--------|------|
| F-TRIP-01 | 创建/编辑/删除旅行，含标题、类型、目的地、起止日期、预算、封面色与图标 | P0 | ✅ |
| F-TRIP-02 | 旅行状态（规划中/进行中/已完成）及自动流转（按日期判断进行中） | P1 | 🟡（手动） |
| F-TRIP-03 | 旅行类型差异化（旅行/差旅/周末使用不同模板与字段） | P1 | ⬜ |
| F-TRIP-04 | 旅行复制（基于已有行程快速新建） | P2 | ⬜ |

**验收标准（F-TRIP-01）**
- 必填项校验（标题、目的地、起止日期），结束日期不得早于开始日期。
- 创建成功后跳转详情页，列表与统计实时更新。

### 6.2 每日规划（P0）

| 编号 | 需求 | 优先级 | 状态 |
|------|------|--------|------|
| F-PLAN-01 | 按天展示，添加行程项（时间、标题、类别、地点、备注、花费） | P0 | ✅ |
| F-PLAN-02 | 行程项打卡完成 | P1 | ✅ |
| F-PLAN-03 | **地图视图**：行程点位标注 + 当天路线连线 + 距离/耗时估算 | **P0** | ⬜ |
| F-PLAN-04 | 行程项**拖拽排序 / 跨天移动** | P1 | ⬜ |
| F-PLAN-05 | **时间冲突检测**（同一时段重叠提醒） | P1 | ⬜ |
| F-PLAN-06 | 地点 **POI 搜索联想**（输入"故宫"自动补全坐标与地址） | P0 | ⬜ |

**验收标准（F-PLAN-03 地图）**
- 选定某天，地图自动聚焦该天所有有坐标的行程点。
- 点位按时间顺序连线，显示相邻点直线距离（后续可升级为真实路线）。
- 无坐标的行程项在地图旁以"未定位"列表提示补充。

### 6.3 预算与费用（P0 / 差旅 P0）

| 编号 | 需求 | 优先级 | 状态 |
|------|------|--------|------|
| F-COST-01 | 记一笔（项目、金额、类别、日期），预算 vs 实际可视化、超支提醒 | P0 | ✅ |
| F-COST-02 | 分类汇总（交通/餐饮/住宿/门票/购物/其他） | P1 | ✅ |
| F-COST-03 | **差旅报销模式**：标记可报销、上传发票图、生成报销汇总表 | P1 | ⬜ |
| F-COST-04 | 多币种支持与汇率换算 | P2 | ⬜ |
| F-COST-05 | 费用导出（Excel / CSV） | P1 | ⬜ |

### 6.4 旅行记录 / 手账（P0）

| 编号 | 需求 | 优先级 | 状态 |
|------|------|--------|------|
| F-NOTE-01 | 图文记录（标题、正文、心情、日期） | P0 | ✅ |
| F-NOTE-02 | **图片上传与展示**（多图、压缩、缩略图） | **P0** | 🟡（字段就绪，前端未做） |
| F-NOTE-03 | 记录关联到具体行程项/地点 | P2 | ⬜ |
| F-NOTE-04 | 记录支持位置标签与天气快照 | P2 | ⬜ |

### 6.5 AI 行程规划（P0，核心卖点）

| 编号 | 需求 | 优先级 | 状态 |
|------|------|--------|------|
| F-AI-01 | 输入目的地/天数/类型/偏好/预算，生成每日行程 | P0 | 🟡（无 Key 时为内置 POI 库示例，未真正调用大模型） |
| F-AI-02 | **接入真实大模型**（OpenAI 兼容），输出与目的地强相关 | **P0** | 🟡（调用链路已就绪，待正式配置 Key 验证） |
| F-AI-03 | 生成结果**一键保存为旅行** | P0 | ✅ |
| F-AI-04 | 局部重新生成（"换掉第 2 天""更轻松一点"） | P1 | ✅ |
| F-AI-05 | 基于真实 POI 库的推荐兜底（无 Key 时也靠谱） | P1 | ✅（内置 24 个热门城市 POI 库） |
| F-AI-06 | AI 行程含预估花费与时间，可直接同步进预算 | P2 | ✅ |

**验收标准（F-AI-02）**
- 同一目的地不同偏好（如"美食优先"vs"亲子"），输出应有明显差异。
- 失败时优雅降级到 POI 规则推荐，并提示用户。

## 7. 信息架构与数据模型

### 7.1 信息架构

```
漫游手账
├── 我的旅行（仪表盘）
│   ├── 旅行卡片墙 / 筛选 / 统计
│   └── 旅行足迹地图（规划中）
├── 旅行详情
│   ├── 每日规划（列表视图 / 地图视图）
│   ├── 预算花销（普通 / 差旅报销模式）
│   └── 旅行记录（手账）
├── AI 规划
├── 灵感发现
└── 我的（账户 / 同步 / 设置，规划中）
```

### 7.2 现有数据模型（v1.0）

```
Trip 1 ──< Activity      （旅行 → 行程项）
Trip 1 ──< Expense       （旅行 → 花销）
Trip 1 ──< Note          （旅行 → 记录）
```

| 实体 | 关键字段 |
|------|---------|
| Trip | id, title, type, destination, startDate, endDate, budget, status, coverColor, coverEmoji |
| Activity | id, tripId, dayDate, startTime, title, category, location, note, cost, done, order |
| Expense | id, tripId, title, category, amount, date |
| Note | id, tripId, title, content, mood, date, images[] |

### 7.3 规划中的数据模型扩展

| 扩展 | 新增字段 / 实体 | 服务的需求 |
|------|---------------|-----------|
| 地图 | Activity 增加 `lat`、`lng`、`address`、`poiId` | F-PLAN-03/06 |
| 用户 | 新增 `User`；各实体增加 `userId` | F-ACC-01/02 |
| 报销 | Expense 增加 `reimbursable`、`invoiceUrl`、`currency` | F-COST-03/04 |
| 图片 | 新增 `Attachment`（或对象存储 URL 列表） | F-NOTE-02 |
| 分享 | 新增 `ShareToken`（tripId, token, 过期时间, 只读） | F-SHARE-01 |
| 灵感 | 新增 `Inspiration` / `Wishlist` | F-DISC-04/05 |

## 8. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 首屏可交互 < 2s；列表/详情接口 P95 < 300ms |
| 兼容 | 移动端优先的响应式（主要使用场景在手机），兼容主流浏览器 |
| 可用性 | 关键操作有 Toast 反馈、错误优雅降级（如 AI 失败兜底） |
| 安全 | 账户密码加密存储；分享链接不可枚举；上传文件类型/大小校验 |
| 隐私 | 个人数据默认私有；不采集敏感信息；提供数据导出与删除 |
| 可维护 | 前后端分层清晰、类型完备；数据库可平滑从 SQLite 切 PostgreSQL |
| 合规 | 第三方内容数据需合法授权，禁止爬取 |

## 9. 当前版本（v1.0）现状与差距

### 9.1 已具备
- 旅行 / 行程 / 预算 / 记录的完整 CRUD 与三件套闭环。
- AI 推荐与灵感发现的**接口骨架**（含 Mock 兜底、Key 可配置）。
- 活泼多彩响应式 UI、本地 SQLite 持久化、可推送 GitHub。

### 9.2 核心差距（按影响排序，随迭代动态更新）
1. ~~**AI 推荐"形似神不似"**~~：已通过内置 POI 库 + 预估花费 + 局部重新生成大幅改善（F-AI-04/05/06 已完成）；仍需正式接入大模型 Key 做最终验证（F-AI-02）。
2. **缺地图**：旅行规划无地图 = 盲规划，P0 缺失。
3. **记录不能传图**：字段就绪但前端缺失，"记录"体验残缺。
4. **类型无差异化**：旅行/差旅/周末仅换图标，差旅报销诉求未满足。
5. **无账户与同步**："后端+数据库"价值未兑现，单机单库。
6. **无分享/导出**：缺少天然传播与协同点。
7. **无出行中视图**：只服务"规划"，忽略"执行"阶段。

## 10. 风险与依赖

| 风险 | 等级 | 应对 |
|------|------|------|
| 第三方内容（小红书等）无合规 API | 高 | 采用自有内容库 / 官方合作 / 合规数据源；当前 Mock 占位 |
| AI 模型调用成本与稳定性 | 中 | 设速率与额度限制；POI 规则推荐兜底；缓存常见目的地结果 |
| 地图服务 Key 与配额（高德/Mapbox） | 中 | 申请正式 Key；控制调用频次；坐标本地缓存 |
| 多用户后数据库压力 | 中 | 升级 PostgreSQL，加索引与连接池 |
| 图片存储成本与安全 | 中 | 接对象存储（OSS/S3）+ 大小/类型校验 + CDN |
